什么是 HTML 表单#

HTML 表单是一个包含特殊交互控件（如文本框、复选框、单选按钮、提交按钮等）的文档区域。用户可以在这些控件中输入或选择数据，然后将这些数据提交（发送）到 Web 服务器进行处理。

核心结构：<form>元素#

表单元素必须被包裹在<form>和</form>标签内。这个标签本身不可见，但它定义了整个表单的作用范围和行为。 <form> 元素有几个至关重要的属性：

  • action: 指定一个 URL 地址。当用户提交表单时，表单收集到的所有数据都将被发送到这个地址进行处理。这通常是服务器上的一个后端脚本（如 Python、PHP、Node.js 文件）。
  • method: 指定发送数据时使用的 HTTP 方法。最常用的有两个值：GET和POST。GET会将表单数据附加在URL的末尾，然后请求该URL，这种方式常用于数据量小且不敏感的场景，如搜索引擎；POST会将表单数据包裹在HTTP请求中发送，这种方式更安全稳定。

常用的表单控件#

  1. <input> <input>元素的具体形态由type属性决定。
    • type="text": 单行文本输入框。
    • type="password": 密码输入框，输入内容会以圆点或星号显示。
    • type="email": 电子邮件输入框。在提交时，浏览器会自动检查其格式是否符合电子邮件规范。
    • type="number": 数字输入框，通常会带有一个小箭头用于增减数值。
    • type="date": 日期选择器。
    • type="checkbox": 复选框，允许用户从多个选项中选择零个或多个。
    • type="radio": 单选按钮，允许用户从多个选项中只选择一个。
    • type="file": 文件选择框，用于上传文件。
    • type="submit": 提交按钮。点击后会触发表单的提交动作，将数据发送到 action 指定的 URL。
    • type="reset": 重置按钮，点击后会将表单内所有控件恢复到初始值。
    • type="button": 普通按钮，默认无任何行为，通常需要配合 JavaScript 来实现自定义功能。 <input>还有一些通用的属性。
    • name: 控件的名字，在提交表单时标识这个控件的数据。
    • id: 元素的唯一标识符。
    • value: 元素的初始值。
    • placeholder: 在输入框为空时显示的文字。
    • required: 布尔值，表示该字段在提交表单前是否必须被填写。
    • disabled: 布尔值，表示该字段被禁用，用户无法与其交互，数据也不会被提交。
    • readonly: 布尔值，表示用户无法与其交互，但其数据会被提交。
  2. <label> <label>元素为表单控件提供文字标签。
  3. <textarea> 用于多行文本输入。
  4. <select>和<option> 用于创建下拉选择菜单。
