[高性能计算] CUDA GEMM 的基本优化

June 28, 2025

在前一篇文章中，我们写了一个用 CUDA 实现的基础 GEMM ，现在我们来尝试优化这个朴素实现。

首先，我们需要知道朴素实现的性能瓶颈在哪里。

__global__ void naiveGEMM(float *d_A, float *d_B, float *d_C, int M, int N, int K) {
    int row = blockIdx.y*blockDim.y+threadIdx.y;
    int col = blockIdx.x*blockDim.x+threadIdx.x;
    if(row < M && col < N) {
        float sum = 0;
        for(int k = 0; k < K; k++) {
            sum += d_A[row*K+k]*d_B[k*N+col];
        }
        d_C[row*N+col] = sum;
    }
}

在朴素实现中，每个线程独立地调用A和B中的相应行列作点乘，需要注意到的是，这些访问的都是 GPU 上的全局内存，访问速度非常慢。同时，不难发现事实上不同的线程重复访问了很多相同的元素，假如我们把这些相同的元素先统一存到共享内存上再去访问，访问速度将会得到数量级上的提升。

具体应该怎么实现呢？一个自然的想法是利用分块矩阵乘法的思想。我们可以将两个矩阵视作分块矩阵，每个块的大小与线程块的大小相同，这样原矩阵乘法就变成了处理一个个分块矩阵乘法。对于每个线程块，我们可以遍历A和B中相应行列的分块矩阵作乘法，对于每个分块矩阵乘法，我们可以让每个线程先将A和B中的一个元素存储到共享内存上，等所有线程都存储完毕后再一起计算分块矩阵乘法，最后累加即可得到结果。

总结一下：

  • 一个线程块中的所有线程协作将A和B中的一个分块矩阵从全局内存加载到共享内存
  • 使用__syncthreads()同步线程
  • 线程块中所有线程分别从共享内存中读取数据计算
  • 使用__syncthreads()同步线程
  • 重复过程直到遍历完所有分块矩阵

__global__ void tiledGEMM(float *d_A, float *d_B, float *d_C, int M, int N, int K) {
    __shared__ float As[TILE_WIDTH][TILE_WIDTH], Bs[TILE_WIDTH][TILE_WIDTH];
    int tx = threadIdx.x, ty = threadIdx.y;
    int row = blockIdx.y*blockDim.y+ty, col = blockIdx.x*blockDim.x+tx;
    float sum = 0;
    for(int p = 0; p < (K+TILE_WIDTH-1)/TILE_WIDTH; p++) {
        // each thread will first store an element of A and an element of B
        int A_row_idx = row, A_col_idx = p*TILE_WIDTH+tx;
        if(A_row_idx < M && A_col_idx < K) {
            As[ty][tx] = d_A[A_row_idx*K+A_col_idx];
        }
        else As[ty][tx] = 0;
        int B_row_idx = p*TILE_WIDTH+ty, B_col_idx = col;
        if(B_row_idx < K && B_col_idx < N) {
            Bs[ty][tx] = d_B[B_row_idx*N+B_col_idx];
        }
        else Bs[ty][tx] = 0;

        __syncthreads();

        for(int k = 0; k < TILE_WIDTH; k++) {
            sum += As[ty][k]*Bs[k][tx];
        }

        __syncthreads();
    }
    if(row < M && col < N) {
        d_C[row*N+col] = sum;
    }
}
