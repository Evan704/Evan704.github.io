C++中的Lambda表达式

June 29, 2025
Table of Contents

Lambda 表达式是 C++11 引入的一项强大特性，它允许你在代码中定义一个匿名的、临时的函数对象，这极大地简化了代码，并一定程度上增强了代码的可读性。

Why Lambda?#

在 C++11 之前，如果想向一个算法传递一个简单的函数（比如向std::sort传递用于比较大小的函数），你需要在全局或类作用域中定义一个独立的函数，或者定义一个函数对象。这两种方法都较为繁琐，且可读性较差（定义和使用的地方间隔很远）。

而利用 Lambda 表达式，你可以就地定义一个简单的函数，使代码更紧凑易读。

基本语法#

c++

[捕获列表](参数列表) mutable（可选） noexcept（可选） -> 返回类型（可选）{

  • []：捕获列表
    • 用于从外在作用域“捕获”变量
  • ()：参数列表
  • mutable：可变说明符
    • 默认情况下，通过值捕获的变量在 Lambda 函数体中是 const 的，不可修改。
  • noexcept：异常说明符
    • 与普通函数相同。
  • ->：返回类型
    • 一般情况下，编译器会根据return语句自动推断返回值类型。 简单的例子：

[]() {
    std::cout << "Hello World!" << std::endl;
}(); // () 表示立即调用
auto fun = []() {
    std::cout << "Hello World!" << std::endl;
}; // 也可以用变量存储

捕获列表#

Lambda 表达式需要从父作用域捕获一些变量，捕获的方式是可以调整的。

捕获方式说明
[]不捕获任何外部变量
[=]按值捕获，函数体中捕获的变量是外部变量的副本，且默认是只读的
[&]按引用捕获，函数体中捕获的变量是外部变量的引用，可以修改
[x, &y]自定义捕获，指定捕获哪些变量，以何种方式捕获
[=, &x]默认值捕获，但是x按引用捕获
[&, y]默认引用捕获，但是y按值捕获
[this]捕获this指针，在类的的成员函数中使用

初始化捕获#

C++14 引入了初始化捕获，允许你在捕获列表中自定义新变量，这些变量只在 Lambda 函数体中可见。

c++
1
2
3
4

int x = 1;
[z = x+1]() {
    std::cout << z << std::endl;
}();

应用场景#

最简单的应用场景是在std::sort()中传递比较函数和用std::for_each()遍历。

#include <iostream>
#include <vector>
#include <algorithm>

int main() {
    std::vector<int> v = {4, 1, 3, 5, 2};
    std::sort(v.begin(), v.end(), [](int a, int b) {
        return a > b;
    });
    int factor = 10;
    std::for_each(v.begin(), v.end(), [factor](int n) {
        std::cout << n * factor << " ";
    });
    // 输出: 50 40 30 20 10
    return 0;
}
