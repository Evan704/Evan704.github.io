如何搭建一个属于自己的博客？一个简单而便捷的博客搭建方案是Hugo+Github Pages，它具有许多优势：

  • 完全免费：不需要购买服务器。
  • 速度极快：Hugo 是目前生成网站最快的工具之一。
  • 高度可控：允许你定制自己博客的内容。

准备工作：下载Scoop与Hugo#

首先，允许 Powershell 执行远程脚本。打开 Powershell ，执行以下命令：

powershell
1

Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Powershell 会询问是否确认，输入 y 回车确认即可。

Hint: -Scope CurrentUser保证了这个设置只对当前用户有效，是安全的做法。

接下来就可以安装 Scoop ，在 Powershell 中执行：

powershell
1

irm get.scoop.sh | iex

如果无法下载，可以直接在浏览器中打开get.scoop.sh，另存为.ps1文件并运行。 安装脚本运行完毕后，可以输入以下命令来验证安装是否成功：

powershell
1

scoop help

现在，你就可以直接使用scoop命令来安装软件了！

在以下教程中，默认你已经安装了 VS Code 和 Git 。

首先我们可以安装增强版下载工具 aria2 (可选但推荐，Scoop会自动调用它进行多线程下载)：

powershell
1

scoop install aria2

然后需要下载Hugo：

powershell
1

scoop install hugo-extended

至此，准备工作完成。

本地搭建你的博客#

1. 创建一个博客项目#

进入你希望存放博客的目录（以E:\MyBlog为例），运行以下命令：

powershell
1
2

hugo new site MyBlog
cd MyBlog

将来，你的博客的文件将会存储在E:\MyBlog下。

2. 安装一个主题#

以一个流行的简洁而功能强大的主题 PaperMod 为例（即本博客使用的主题）。 首先，将博客初始化为 Git 仓库，在E:\MyBlog下执行：

powershell
1

git init

接着，安装 PaperMod

powershell
1

git submodule add https://github.com/adityatelange/hugo-PaperMod.git themes/PaperMod

3. 配置你的博客#

在 VS Code中，打开根目录下的hugo.toml文件。在这里，你可以配置你的博客。 为了实现基础功能，将该文件修改如下：

toml
1
2
3
4
5
6

baseURL = "https://your-github-username.github.io/" # 重要！将你的 github 用户名填入！
languageCode = "zh-cn" # 网站语言
title = "My Blog" # 网站标题
theme = "PaperMod" # 使用的主题

# 将来的配置写在下面

Hint : 务必修改 baseURL 中的网址！

开始写第一篇文章！#

运行以下命令创建第一篇文章：

powershell
1

hugo new content posts/test.md

用该命令创建新文件的好处是新创建的文件会带有默认的 Front Matter：

toml
1
2
3
4
5

+++
date = '2025-06-25T09:47:11+08:00'
draft = false
title = 'Test'
+++

Front Matter 中可以设置文章的一些基本信息，如标题，日期，是否为草稿等。

在 Front Matter 下方，你就可以开始书写你的正文了！写完保存后，你就可以执行以下命令来预览你的第一篇文章！

powershell
1

hugo server -D # -D 参数意为显示草稿文章，默认不显示。

如果一切正常，终端会提示： Web Server is available at http://localhost:1313/

此时，在浏览器中打开http://localhost:1313/即可预览你的博客。

你在本地对文章进行修改时，预览版网页也会动态地显示修改，十分方便。

将博客部署到网络#

1. 在 Github 新建两个仓库#

一个用于保存源码（假设仓库名为BlogSource），可以设置为 Private ；另一个用于保存发布的静态网页，必须设置为 Public ，且仓库名必须为<username>.github.io。

2. 推送源码到 Github#

在博客的根目录下执行：

powershell
1
2
3
4

git remote add origin git@github.com:<username>/BlogSource.git # 若设置了 SSH
git add .
git commit -m "Initialize"
git push -u origin master

3. 生成并部署网站#

在博客根目录下执行：

powershell
1

hugo

这会生成一个 public 文件夹，里面就是最终的网页文件。

进入 public 文件夹，并初始化为 Git 仓库，然后关联到发布仓库。

powershell
1
2
3
4
5
6

cd public
git init
git remote add origin git@github.com:<username>/<username>.github.io.git
git add .
git commit -m "Build web"
git push -u origin master

等待几分钟，就可以访问https://your-github-username.github.io并查看你的博客了！

优化发布流程#

现在每次写完文章都需要手动hugo再cd public再推送，有点麻烦。我们可以利用 Github Action 实现自动化部署。

1. 设置 workflow#

首先，在博客根目录下新建.github文件夹，再在其中新建workflow文件夹。接着在里面新建deploy.yml，并用 VS Code 打开，贴入以下内容：

name: Deploy Hugo site to Pages

on:
  push:
    branches:
      - master # 监视 master 分支

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          submodules: true  # 拉取主题 submodule

      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: 'latest'
          extended: true

      - name: Build
        run: hugo --minify

      - name: Deploy
        uses: peaceiris/actions-gh-pages@v4
        with:
          deploy_key: ${{ secrets.ACTIONS_DEPLOY_KEY }}
          publish_dir: ./public
          # 部署到 <username>.github.io 仓库
          external_repository: <username>/<username>.github.io
          publish_branch: master
          user_name: 'github-actions[bot]'
          user_email: 'github-actions[bot]@users.noreply.github.com'

接下来我们需要给这个 Github Action 设置 deploy key。

2. 生成 SSH 密钥#

在任意路径下运行以下命令：

powershell
1

ssh-keygen -t ed25519 -C "hugo deploy key" -f ./hugo-deploy-key

终端会提示Enter passphrase (empty for no passphrase):，此时需直接输入回车。完成后，在当前路径下会得到两个新文件：

  • hugo-deploy-key（私钥）
  • hugo-deploy-key.pub（公钥）

3. 配置公钥#

复制公钥中所有内容，打开发布仓库（<username>.github.io仓库），在 Settings 选项卡中选择 Deploy Keys ，点击 Add deploy key 。

随意输入一个 Title （你看得懂就行），将复制内容贴入 Key 中，勾选 Allow write access ，然后点击 Add key 。

4. 配置私钥#

复制私钥中所有内容，打开源码仓库（BlogSource仓库），在 Settings 选项卡中选择 Secrets and variables ，选择 Actions 。点击 New repository secret ，在 Name 中输入 ACTION_DEPLOY_KEY，在 Secret 中粘贴复制内容，点击 Add secret。

至此，可以重新向源码仓库提交更改，此时 workflow 已经配置完成。

在未来写新文章时的流程如下：

  1. hugo new content posts/something_new.md
  2. 写完后保存，将 Front Matter 中的 draft 属性修改为 false 。
  3. 通过hugo server进行预览。（重要，也可直接使用hugo）
  4. 向源码仓库推送，workflow 会自动完成剩余的工作。

怎么实现 tags 功能#

非常简单，完全不需要额外的操作，只需在文章的 Front Matter 中添加 tags 属性即可，例如：

toml
1
2
3
4

+++
# 其他属性
tags = ["Code"]
+++

即可为这篇文章创建一个名为 Code 的 tag。点击这个 tag 还能直接查看所有含有这个 tag 的文章。
