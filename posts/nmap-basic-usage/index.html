[网络安全] nmap 基础

July 1, 2025

nmap 的各个参数用法可以参考nmap -h的输出。下面对常用的一些参数进行介绍。

  • -p: 后面接扫描端口范围。
    • nmap -p 80,443 <目标IP>: 只扫描端口80和443。
    • nmap -p 1-100 <目标IP>: 扫描1到100号端口。
    • nmap -F <目标IP>: 快速扫描，只扫描Nmap中列出的最常见的100个端口。
    • nmap -p- <目标IP>: 扫描所有65535个端口，非常耗时。
  • -sn: Ping 扫描，只进行主机发现，不扫描端口，速度很快。
  • -sT: TCP 扫描，尝试对目标主机的指定端口范围内的所有 端口完整进行TCP三次握手。若握手成功，则端口开放，记录为open。若收到RST，则说明端口关闭，记录为closed。若未收到回复，则一般是被防火墙拦截，记录为filtered。这种扫描方式较为可靠，但是会被防火墙或 IDS 记录下来。
  • -sS: SYN 扫描（半开扫描），顾名思义，只向对应端口发送一个SYN包，对结果处理同 TCP 扫描。这种扫描一般不会被防火墙和 IDS 记录，且速度比 TCP 扫描更快，但是需要 sudo 权限（这也是 nmap 的默认扫描方式）。
  • -sU: UDP 扫描。与前两种扫描方式不同的是，无论端口是开放还是被防火墙保护了，都不会收到回复，所以这种情况会被记录为open|filtered。而当向关闭的端口发送 UDP 包时，一般会收到一个 ICMP（ping）包表明该端口已关闭，这种情况会被记录为closed。

除了上述几种扫描方式外，还有-sN,-sF,-sX等扫描方式，但是它们相对少见，因此没有介绍。

  • -sV: 扫描具体的服务名称和版本号。
  • -O: 扫描目标主机使用的是哪个操作系统。
  • -T0到-T5: 设置扫描速度，数字越大越快，也越容易被发现。
